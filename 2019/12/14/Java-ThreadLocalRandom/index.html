Java-ThreadLocalRandom

  • Java

Random生成随机数

需要根据老的种子来生成新的种子，然后根据新的种子来生成随机数，但是在多线程条件下可能会有多个线程使用一个老的种子来更新，为了保证原子性，在第一个线程生成新的种子后，第二个线程需要丢弃自己的老种子来使用第一个线程的新种子，后续的线程也是。在多个线程拿到新的种子去更新老的种子时，使用CAS方式更新，这时只有一个线程能够用获取到，其他线程需要通过循环来获取，这样就会有大量的自旋重试，会降低并发性。

ThreadLocalRandom

随后ThreadLocalRandom进行了改进，它并没有使用继承自Random的原子性种子变量，具体的种子放在了调用线程的threadLocalRandomSeed变量中，这样在进行更新时，根据自己线程中维护的种子，不会产生多线程竞争原子变量问题。
