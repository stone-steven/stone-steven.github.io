Java-泛型

  • Java

概述

什么是泛型

泛型就是类型参数化，处理的数据类型是不固定的，而可以作为参数传入。

基本原理

Java虚拟机没有泛型类对象，会将泛型类代码转换为普通的非泛型代码，将类型参数消除，换成限定类型（无限定类型则用Object）。

优点

  • 更好的安全性

    使用Object在代码写错的时候开发环境和编译器不能帮我们发现问题。而使用泛型时，编译器可以根据类型参数检查，避免使用了错误的类型。

  • 更好的可读性

    可以很清楚的知道是什么类型。

使用

泛型方法

除了泛型类之外，方法也可以是泛型的，这与是否是泛型类无关，

泛型接口

接口也可以是泛型的。

类型参数的限定

可以为类型参数指定一个上界,限定类型后如果使用错误编译器会有提示，类型擦除时候就不会指定为Object，而会指定为边界类型。

通配符

?表示有限定通配符，匹配E或者E的某个子类型。 ### 和对比 1. 用于**定义**类型参数，声明了一个参数T，可以放在泛型限定类名后面，泛型方法返回值前面。 2. 用于**实例化**类型参数，用于实例化泛型变量中的类型参数，具体类型是未知的，只知道他是E或者E的子类型。 ### 和对比 1. 他们的目的都是使得方法接口更加灵活，可以接受更广泛的类型 2. 用于灵活写入或比较，使得对象可以写入父类型的容器，使得父类型的比较方法可以应用于子类对象，他不能被类型参数形式替代。可以赋值给任何T以及T的父类的集合，下界为T。它也可以进行get操作，但是类型会丢失，只能返回Object对象。 3. 和

用于灵活方式读取，他们可以使用类型参数的形式替代。<? extends T>可以赋值给任何T以及T的子类的集合，上界为T，取出来的泛型带有泛型限制，向上转型为T，除了null外任何元素都不能添加进<? extends T>集合内。它可以返回带类型的元素，但是只能返回T以及T的父类的对象，因为子类型被擦除了。

通配符和类型参数的选择

  1. 通配符形式的都可以用类型参数代替，通配符能做的类型参数都能做。
  2. 通配符形式可以减少类型参数，形式上更为简单，可读性也更好。
  3. 如果类型参数之间有依赖关系，或者返回值依赖类型参数，或者需要写操作，必须需要类型参数。

泛型类局限性

  1. 不能用基本类型实例化类型参数

  2. 运行时类型查询只适用于原始类型，判断泛型将返回错误。

  3. 不能创建参数化类型数组

  4. 不能实例化类型变量，不能使用形如new T()这样的方式。可以使用构造器表达式实现。
